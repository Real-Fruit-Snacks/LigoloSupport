// architecture

Download, configure, tunnel.

The auto command orchestrates the full workflow: fetch binaries, create TUN interface, serve agents, display instructions, and start the proxy.

01

Binary Download

Fetches the latest ligolo-ng release tag from GitHub API. Downloads proxy (linux/amd64) and agents for linux/amd64, linux/arm64, windows/amd64, and darwin/arm64. Extracts and sets execute permissions.

02

TUN Interface

Creates a TUN interface via ip tuntap, brings it up, and validates the link state. Skips creation if the interface already exists. Teardown removes the interface cleanly.

03

Agent Transfer

Starts python3 HTTP server on port 8000 serving the binary directory. Generates platform-specific download commands with your detected IP. Supports curl, iwr, and certutil.

04

Proxy + Routing

Launches ligolo-proxy with self-signed cert on the configured port. After agent connection and session selection, add-route creates ip routes through the TUN device for target network access.

// quick start

Setup in seconds.

Download the script, run auto, follow the steps. Access the target network directly without proxychains.

setup.sh 
# Download and run
$ curl -O https://raw.githubusercontent.com/
    Real-Fruit-Snacks/LigoloSupport/main/
    ligolo-helper.sh
$ chmod +x ligolo-helper.sh

# Full auto-setup
$ sudo ./ligolo-helper.sh auto

# When done
$ sudo ./ligolo-helper.sh cleanup
workflow.sh 
# After agent connects to proxy
ligolo> session
  [0] YOURPC\user @ YOURPC

ligolo> ifconfig
  Interface 3: Ethernet0
    10.10.10.5/24

# Add route (new terminal)
$ sudo ./ligolo-helper.sh add-route 10.10.10.0/24

# Start tunnel (proxy console)
ligolo> tunnel_start --tun ligolo
  INFO Tunnel started on interface ligolo
